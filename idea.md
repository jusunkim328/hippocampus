# Elasticsearch Agent Builder Hackathon - Ideas

## 1. Serendipity Engine (우연한 발견 엔진)

완전히 다른 분야의 논문/특허/뉴스를 ES에 인덱싱하고, **의도적으로 분야 간 예상치 못한 연결고리**를 찾아주는 에이전트

### 컨셉
- "접착제 연구" ↔ "도마뱀 발바닥 논문" → "게코 테이프 발명"
- 시맨틱 유사성으로 **서로 모르는 분야 사이의 교차점**을 발견

### 워크플로우
1. 사용자가 자기 연구/관심사를 입력
2. 에이전트가 완전히 다른 도메인에서 유사 패턴 검색
3. 교차 분야 연결 근거 분석
4. "이 생물학 논문이 당신의 건축 프로젝트에 영감을 줄 수 있습니다" 같은 추천 생성

### 강점
- "검색"의 본질을 뒤집음 — 찾고 싶은 걸 찾는 게 아니라, **몰랐던 걸 발견**
- 검색 엔진 회사 해커톤에서 검색 패러다임 자체를 뒤집는 의외성
- ES의 시맨틱 검색 + 하이브리드 검색 + 리랭킹 전부 활용
- 학술/R&D/스타트업 등 실제 수요 존재

### 기술 스택
- Elasticsearch: ELSER 시맨틱 검색, RRF 하이브리드 검색, 시맨틱 리랭킹
- 데이터: arXiv 논문, 특허 데이터, 뉴스 아티클 (오픈 데이터)
- LLM: 교차 분야 연결 설명 및 인사이트 생성

## 2. Cold Case Detective (미제사건 탐정 에이전트)

실제 공개된 미제사건 데이터를 ES에 인덱싱하고, 사건 간 패턴 매칭, 증거 상관분석, 새로운 수사 방향을 제안하는 에이전트

### 컨셉
- 사건 설명 입력 → 유사 수법의 다른 사건 시맨틱 검색
- 시간/장소/방법 다차원 상관분석
- "이 3건의 사건이 연결될 가능성이 있습니다" + 근거 제시

### 워크플로우
1. 미제사건 데이터 수집 및 ES 인덱싱 (공개 데이터셋)
2. 사용자가 사건 정보 입력 또는 특정 사건 선택
3. 에이전트가 유사 패턴 사건 시맨틱 검색
4. 시간/장소/수법 기반 다차원 상관분석
5. 새로운 수사 가설 및 방향 제안

### 강점
- 데모가 극적이고 스토리텔링이 강력 (넷플릭스 다큐 느낌)
- 데모 품질 심사 기준(30%)에서 압도적 우위
- 시맨틱 검색의 "숨겨진 연결 찾기" 능력을 극적으로 시연

### 기술 스택
- Elasticsearch: 시맨틱 검색, 지리공간 쿼리, 다차원 필터링
- 데이터: 공개 범죄 데이터셋, 뉴스 아카이브
- LLM: 상관관계 분석, 수사 가설 생성

## 3. Time Warp News (시간여행 뉴스 에이전트)

오늘의 뉴스를 넣으면, 역사 속 비슷한 사건을 찾아서 **"역사가 반복되고 있다"는 패턴**을 보여주는 에이전트

### 컨셉
- 오늘 뉴스: "AI가 일자리를 대체한다"
- 에이전트 → 1800년대 산업혁명 러다이트 운동, 1950년대 자동화 공포 기사 시맨틱 매칭
- "그때는 이렇게 끝났고, 이번에는 이런 점이 다릅니다" 비교 분석 생성

### 워크플로우
1. 역사 뉴스/사건 데이터 ES 인덱싱 (수십~수백 년 범위)
2. 사용자가 현재 뉴스 URL 또는 토픽 입력
3. 에이전트가 핵심 테마 추출
4. 역사 속 시맨틱 유사 사건 다단계 검색
5. 시대별 비교 분석 + "역사적 교훈" 리포트 생성

### 강점
- 누구나 공감, 교육적 가치
- 시각적 타임라인 데모가 매력적
- 시맨틱 검색이 시간 축을 넘나드는 강력한 활용 사례

### 기술 스택
- Elasticsearch: 시맨틱 검색, 시간 기반 필터링, RRF 하이브리드 검색
- 데이터: 역사 뉴스 아카이브 (오픈 데이터), 위키피디아 역사 이벤트
- LLM: 테마 추출, 시대 간 비교 분석, 교훈 도출

## 4. Dream Weaver (꿈 분석 에이전트)

사용자가 매일 꿈 일기를 기록하면 ES에 인덱싱하고, 반복되는 상징, 감정 패턴, 꿈 간 연결을 분석하는 에이전트

### 컨셉
- "또 바다에서 쫓기는 꿈을 꿨어"
- 에이전트 → 과거 꿈들에서 "물" "추격" 테마 시맨틱 검색 → 빈도/감정 추이 분석
- 심리학 문헌 기반 해석 + 개인 패턴 리포트 생성

### 워크플로우
1. 심리학/꿈 해석 문헌 + 사용자 꿈 일기 ES 인덱싱
2. 사용자가 오늘의 꿈 기록 입력
3. 에이전트가 상징/감정/테마 자동 추출
4. 과거 꿈들과 시맨틱 패턴 매칭
5. 시간에 따른 감정 추이 분석
6. 심리학 문헌 기반 해석 + 개인 패턴 리포트 생성

### 강점
- 개인적이고 감성적, 완전히 예상 밖의 ES 활용
- "검색 엔진으로 내면을 탐색한다"는 시적인 내러티브
- 데모에서 실제 꿈 일기 → 분석 과정이 인상적

### 기술 스택
- Elasticsearch: 시맨틱 검색, 어그리게이션(감정/테마 빈도), 시계열 분석
- 데이터: 꿈 해석 심리학 문헌 (오픈 데이터), 샘플 꿈 일기
- LLM: 상징 추출, 감정 분류, 심리학적 해석 생성

## 5. Deja Vu Detector (데자뷔 탐지기)

"이 아이디어, 이미 누가 했다." — 스타트업 아이디어/사업계획을 넣으면 **이미 시도되었던 유사 프로젝트**를 찾아주는 에이전트

### 컨셉
- 실패한 스타트업 데이터(CrunchBase, 공개 포스트모템) + 특허 + 킥스타터 프로젝트를 ES에 인덱싱
- 사용자: "AI로 반려동물 건강 모니터링 앱 만들 거야"
- 에이전트 → 유사 시도 5건 발견 → 실패 원인 분석 → "이 3가지를 피하면 성공 확률이 높습니다"

### 워크플로우
1. 스타트업 포스트모템, 특허, 킥스타터/프로덕트헌트 데이터 ES 인덱싱
2. 사용자가 아이디어/사업 계획 입력
3. 에이전트가 핵심 컨셉 추출 → 유사 프로젝트 시맨틱 검색
4. 성공/실패 사례 분류 및 원인 분석
5. "현실 체크" 리포트 생성: 유사 시도, 실패 원인, 차별화 전략 제안

### 강점
- 검색을 "영감"이 아니라 **"현실 체크"**에 쓴다는 반전
- 창업자에게 실제로 돈이 되는 도구, 실용적 가치 극대
- 데모가 직관적이고 임팩트 강함

### 기술 스택
- Elasticsearch: 시맨틱 검색, 어그리게이션(성공/실패 분류), 유사도 스코어링
- 데이터: 공개 스타트업 포스트모템, 킥스타터/프로덕트헌트 데이터, 특허 데이터
- LLM: 아이디어 분해, 실패 원인 분석, 차별화 전략 생성

## 6. Cultural Bridge (문화 가교 에이전트)

서로 다른 문화권의 개념/속담/관용어를 **시맨틱으로 매칭**하는 에이전트

### 컨셉
- 전 세계 속담, 관용어, 문화 고유 개념을 ES에 인덱싱
- "한국어 '눈치'와 의미적으로 가장 가까운 일본/독일/브라질 개념은?"
- 에이전트 → 시맨틱 검색으로 문화 간 개념 매핑 → 미묘한 차이점 분석 → "번역 불가능한 단어" 간의 관계 지도 생성

### 워크플로우
1. 전 세계 속담, 관용어, 문화 고유 개념 + 설명을 ES에 인덱싱
2. 사용자가 특정 문화 개념 또는 속담 입력
3. 에이전트가 의미 추출 → 다른 문화권에서 시맨틱 유사 개념 검색
4. 문화 간 미묘한 차이점/공통점 분석
5. "번역 불가능한 단어" 관계 지도 + 문화적 맥락 리포트 생성

### 강점
- 시맨틱 검색이 **언어의 벽을 넘는** 순간을 보여줌
- 감동적이고 교육적, 글로벌 심사위원에게 강력 어필
- 시맨틱 검색의 "의미 기반 매칭" 본질을 가장 직관적으로 시연

### 기술 스택
- Elasticsearch: 다국어 시맨틱 검색, 벡터 유사도, 어그리게이션
- 데이터: 위키피디아 문화 항목, 다국어 속담 DB, 번역 불가능 단어 사전 (오픈 데이터)
- LLM: 문화적 맥락 해석, 차이점 분석, 관계 설명 생성

## 7. Ripple Effect (나비효과 시뮬레이터)

"만약 X가 일어나면?" — 과거 유사 사건의 연쇄 반응을 추적해 **미래 파급효과를 예측**하는 에이전트

### 컨셉
- 역사적 사건과 그 후속 결과 데이터를 인과관계 체인으로 ES에 인덱싱
- 사용자: "미국이 관세를 올리면?"
- 에이전트 → 과거 관세 인상 사건 검색 → 후속 연쇄 반응 추적(무역량 감소 → 물가 상승 → 소비 위축 → ...) → 3단계 파급효과 예측 리포트

### 워크플로우
1. 역사적 사건 + 인과관계 체인 데이터를 ES에 인덱싱
2. 사용자가 "만약 X가 일어나면?" 질문 입력
3. 에이전트가 X와 유사한 과거 사건 시맨틱 검색
4. 각 사건의 후속 연쇄 반응(1차 → 2차 → 3차 효과) 추적
5. 파급효과 예측 리포트 + 인과관계 다이어그램 생성

### 강점
- 검색을 **시간의 인과관계를 따라가는 도구**로 변환
- 정책 입안자, 투자자, 기자 모두에게 유용
- 1번(공간 축 연결)과 대비되는 "시간 축 연결"로 차별화

### 기술 스택
- Elasticsearch: 시맨틱 검색, 중첩 문서(인과 체인), 시간 기반 필터링
- 데이터: 위키피디아 역사 이벤트, 경제 지표 데이터, 뉴스 아카이브 (오픈 데이터)
- LLM: 인과관계 추론, 파급효과 예측, 시나리오 생성

## 8. Mood Cartographer (감정 지도 제작자)

전 세계 문학 작품 속 감정 표현을 인덱싱하고, 사용자의 복잡한 감정을 **가장 정확하게 표현한 문학 구절**을 찾아주는 에이전트

### 컨셉
- 수만 편의 소설/시/에세이에서 감정 표현 구절을 ES에 인덱싱
- 사용자: "뭔가 슬프진 않은데 공허하고, 그리운 건 아닌데 무언가가 빠진 느낌"
- 에이전트 → 시맨틱 검색으로 가장 가까운 감정 표현 구절 발견 → 무라카미 하루키의 한 문단 → "이 감정은 포르투갈어로 'saudade'라고 합니다"

### 워크플로우
1. 문학 작품 속 감정 표현 구절 + 메타데이터를 ES에 인덱싱
2. 사용자가 자신의 감정을 자유롭게 서술
3. 에이전트가 감정 벡터 추출 → 가장 가까운 문학 구절 시맨틱 검색
4. 여러 문화/언어권의 감정 단어 매칭
5. "당신의 감정에 가장 가까운 표현들" 리포트 생성

### 강점
- 시맨틱 검색의 본질("의미의 근접성")을 가장 아름답게 보여줌
- 소셜 바이럴 가능성 극대 — 사람들이 자기 감정 결과를 공유하고 싶어함
- 감성적이고 시적인 데모

### 기술 스택
- Elasticsearch: 시맨틱 검색, 벡터 유사도 스코어링, 다국어 인덱싱
- 데이터: Project Gutenberg 문학 작품, 시 아카이브, 번역 불가능 감정 단어 사전 (오픈 데이터)
- LLM: 감정 분석, 문학적 맥락 해석, 감정 단어 매칭

## 9. Ancestor Echo (선조의 메아리)

역사적 인물들의 생애 데이터를 인덱싱하고, **"지금 나와 가장 비슷한 상황에 있었던 역사적 인물"**을 찾아 그들의 선택과 결과를 보여주는 에이전트

### 컨셉
- 역사적 인물 수천 명의 전기/결정적 순간/선택과 결과를 ES에 인덱싱
- 사용자: "30대, 안정적 직장을 그만두고 창업을 고민 중"
- 에이전트 → 비슷한 갈림길에 섰던 인물 검색 → 잭 마, 헨리 포드, 코코 샤넬의 유사 시점 비교 → 각자의 선택과 결과 분석 → "역사가 당신에게 보내는 조언"

### 워크플로우
1. 역사적 인물 전기 데이터 (결정적 순간, 선택, 결과) ES 인덱싱
2. 사용자가 자신의 현재 상황/고민 입력
3. 에이전트가 상황 벡터 추출 → 유사 인생 갈림길 시맨틱 검색
4. 매칭된 인물들의 선택과 그 결과 비교 분석
5. "역사가 당신에게 보내는 조언" 리포트 생성

### 강점
- 개인적이면서 보편적 — "검색으로 인생 상담"이라는 예상 밖 활용
- 데모에서 감정적 몰입감 높음, 스토리텔링 강력
- 역사 데이터의 새로운 활용법을 제시

### 기술 스택
- Elasticsearch: 시맨틱 검색, 구조화 필터링(시대/분야/상황), 유사도 스코어링

## 15. Taste Translator (취향 번역기)

한 도메인의 취향을 완전히 다른 도메인으로 **"번역"**하는 에이전트. 음악 취향 → 와인, 영화 취향 → 건축, 패션 취향 → 수학적 구조까지.

### 컨셉
- 각 도메인의 작품/대상을 스타일 벡터로 인덱싱 (복잡성, 미니멀리즘, 대칭성, 긴장감 등)
- 사용자: "미니멀리즘 가구를 좋아해요"
- 에이전트 → 음악에서 바흐 골드베르크 변주곡, 건축에서 안도 다다오, 수학에서 오일러 공식의 "같은 결"을 찾아 추천
- "당신의 취향 DNA는 '절제된 복잡성'입니다"

### 워크플로우
1. 음악/건축/문학/와인/수학/패션 등 각 도메인의 작품 + 스타일 메타데이터를 ES에 인덱싱
2. 사용자가 좋아하는 것(취향)을 입력
3. 에이전트가 취향의 핵심 속성 벡터 추출
4. 완전히 다른 도메인에서 동일 속성 벡터를 가진 대상 시맨틱 검색
5. "취향 번역 리포트" 생성: 도메인별 추천 + 공통 DNA 설명

### 강점
- **바이럴 잠재력 극대** — "내 음악 취향을 와인으로 번역하면?" SNS 공유 본능 자극
- 검색으로 "취향의 본질"을 추출한다는 철학적 깊이
- 데모가 직관적이고 재미있음

### 기술 스택
- Elasticsearch: 시맨틱 검색, 다차원 벡터 유사도, 크로스 도메인 인덱싱
- 데이터: 음악 리뷰(MusicBrainz), 와인 리뷰, 건축 DB, Project Gutenberg 문학 (오픈 데이터)
- LLM: 취향 속성 추출, 도메인 간 연결 설명, 취향 DNA 분석

## 16. Question Archaeologist (질문 고고학자)

**답을 검색하는 게 아니라, "물어봤어야 할 질문"을 검색한다.** 역사적 돌파구를 만든 건 답이 아니라 질문이었다. 사용자의 문제에 대해 "지금 물어봐야 할 진짜 질문"을 찾아주는 에이전트.

### 컨셉
- 과학/경영/예술 등 분야의 "돌파구가 된 질문"을 맥락과 함께 인덱싱
- 사용자: "우리 팀 생산성이 낮아요"
- 에이전트 → 유사 문제의 돌파구 질문 검색 → "도요타는 'Why를 5번 물었고', 넷플릭스는 '규칙이 문제 아닌가?'를 물었습니다. 당신이 물어야 할 질문은..."

### 워크플로우
1. 역사적 돌파구 질문 + 맥락(문제 상황, 분야, 결과) 데이터를 ES에 인덱싱
2. 사용자가 현재 겪고 있는 문제/상황 입력
3. 에이전트가 문제의 핵심 구조 추출
4. 유사 문제 상황에서 돌파구가 된 질문들 시맨틱 검색
5. 질문 리프레이밍 리포트 생성: 발견된 질문 + 맥락 + "당신에게 적용하면" 제안

### 강점
- 검색 엔진의 존재 이유(=답 찾기)를 정면으로 뒤집는 **메타적 의외성**
- 실용성 극대 — 컨설팅, 연구, 경영에서 즉시 활용
- "좋은 질문이 좋은 답보다 중요하다"는 보편적 공감

### 기술 스택
- Elasticsearch: 시맨틱 검색, 구조화 필터링(분야/시대/문제유형), 유사도 스코어링
- 데이터: 위키피디아 과학사, 경영 사례 연구, TED 토크 트랜스크립트 (오픈 데이터)
- LLM: 문제 구조 분해, 질문 리프레이밍, 맥락 적용 설명

## 17. Idea Necromancer (아이디어 강령술사)

**시대를 너무 앞서가서 죽은 아이디어를 되살린다.** 과거에 발명됐지만 기술 한계로 실패/잊혀진 아이디어 중, 지금의 기술로 드디어 실현 가능한 것을 찾아주는 에이전트.

### 컨셉
- 폐기 특허, 실패한 프로젝트, 중단된 연구를 인덱싱
- 사용자: "AR 글래스 사업을 구상 중"
- 에이전트 → "1968년 Sutherland의 'Sword of Damocles', 2001년 MicroOptical, 2013년 Google Glass... 이들이 실패한 이유와 지금은 가능한 이유를 분석하면..."
- 망한 것들의 무덤에서 보물을 발굴

### 워크플로우
1. 만료 특허, 실패 프로젝트 포스트모템, 중단된 연구 논문을 ES에 인덱싱
2. 사용자가 현재 관심 기술/아이디어 입력
3. 에이전트가 핵심 컨셉 추출 → 과거 유사 시도 시맨틱 검색
4. 각 시도의 실패 원인 분석 + 현재 기술로 해결 가능 여부 판단
5. "부활 가능한 아이디어" 리포트: 과거 시도들, 실패 원인, 현재 기술 매칭, 실현 전략

### 강점
- **"실패의 데이터베이스"**라는 발상 자체가 의외
- 특허 전략, R&D, 창업에 실질적 상업 가치
- 데모 스토리텔링 강력 — "이 1990년 특허가 사실 TikTok을 설명하고 있었습니다"

### 기술 스택
- Elasticsearch: 시맨틱 검색, 시간 기반 필터링, 기술 키워드 어그리게이션
- 데이터: USPTO 만료 특허, 킥스타터 실패 프로젝트, arXiv 중단 연구 (오픈 데이터)
- LLM: 실패 원인 분석, 현재 기술 매칭, 부활 전략 생성

## 18. Anti-Recommendation Engine (안티 추천 엔진)

**"이건 절대 안 좋아하실 건데, 한번 보세요"** — 필터 버블을 의도적으로 깨는 에이전트. 당신의 취향과 **가장 먼 곳**에서, 하지만 **한 가지 예상치 못한 접점**이 있는 콘텐츠를 추천.

### 컨셉
- 사용자의 취향 프로필 분석 → 벡터 공간에서 최대 거리 검색
- 하지만 완전히 랜덤이 아님 — 하나의 숨겨진 공통 차원을 찾아 연결
- 헤비메탈 팬에게 → "이 바로크 하프시코드 곡을 들어보세요. 당신이 좋아하는 메탈리카와 완전히 다르지만, 둘 다 '반복적 긴장-해소 구조'를 공유합니다"

### 워크플로우
1. 다양한 도메인의 콘텐츠를 다차원 속성 벡터로 ES에 인덱싱
2. 사용자가 좋아하는 것들을 입력
3. 에이전트가 취향 벡터 추출 → 벡터 공간에서 최대 거리 + 최소 1개 접점 검색
4. 숨겨진 공통점 분석
5. "안티 추천" 리포트: 추천 콘텐츠 + "왜 이걸 추천했는지" 숨겨진 연결 설명

### 강점
- 추천 시스템의 정반대 — "최대 거리 + 최소 접점" 알고리즘이라는 기술적 독창성
- "왜 이걸 추천한 거지?" → 숨겨진 연결 발견의 쾌감
- 에코체임버/필터버블 시대에 시의적절한 메시지

### 기술 스택
- Elasticsearch: 벡터 공간 최대 거리 검색, 차원별 유사도 필터링, 시맨틱 검색
- 데이터: 음악/영화/도서 리뷰 데이터, 콘텐츠 메타데이터 (오픈 데이터)
- LLM: 취향 벡터 추출, 숨겨진 공통점 발견, 추천 내러티브 생성

## 19. Déjà Vécu (데자뷔쿠 — 이미 살아본 느낌)

**내 일상의 한 순간과 가장 비슷한 영화/소설/역사 장면을 찾는다.**

### 컨셉
- 영화 시놉시스, 소설 장면, 역사 속 일기/편지를 감정+상황 벡터로 인덱싱
- 사용자: "오늘 비 오는 날 카페에서 혼자 커피를 마시며 옛 친구를 생각했다"
- 에이전트 → 영화 《비포 선셋》의 한 장면, 무라카미 하루키의 한 구절, 1923년 파리의 한 일기를 매칭
- "당신의 오늘은 이 장면들과 닮아 있습니다"

### 워크플로우
1. 영화 장면 설명, 소설 주요 구절, 역사 일기/편지를 ES에 인덱싱
2. 사용자가 자신의 일상 순간을 자유롭게 서술
3. 에이전트가 감정+상황+분위기 벡터 추출
4. 영화/소설/역사에서 가장 가까운 장면 시맨틱 검색
5. "오늘의 장면 매칭" 리포트: 매칭된 장면들 + 연결 설명

### 강점
- 내 평범한 일상이 예술/역사와 연결되는 **마법 같은 순간**
- SNS 공유 본능 극대 — "내 오늘이 어떤 영화 장면이야?"
- 시맨틱 검색의 "감정 이해" 능력을 가장 감성적으로 보여줌

### 기술 스택
- Elasticsearch: 시맨틱 검색, 감정+상황 다차원 벡터 유사도
- 데이터: IMDB 시놉시스, Project Gutenberg 문학, 공개 역사 일기 (오픈 데이터)
- LLM: 감정/상황 추출, 장면 매칭 설명, 내러티브 연결

## 20. Argument Alchemist (논증 연금술사)

**당신의 확신을 가장 우아하게 반박할 논거를, 예상 밖의 분야에서 찾는다.**

### 컨셉
- 다분야 연구, 사례, 실험 결과를 ES에 인덱싱
- 사용자: "원격근무가 무조건 좋다"
- 에이전트 → 주장의 전제를 분해 → 각 전제에 대한 반례를 예상 밖 분야에서 검색
- 잠수함 승조원 심리학, 수도원 공동체 연구, 우주비행사 격리 실험에서 반박 근거 발견
- "밀폐 환경에서의 우연한 마주침이 창의성에 미치는 영향..."

### 워크플로우
1. 학술 논문, 실험 결과, 역사적 사례를 다분야로 ES에 인덱싱
2. 사용자가 자신의 주장/확신 입력
3. 에이전트가 주장의 핵심 전제 분해
4. 각 전제에 대해 **다른 분야에서** 반례/반박 근거 시맨틱 검색
5. "최고 품질의 반박" 리포트: 반박 근거 + 출처 + "그럼에도 당신의 주장이 맞을 조건"

### 강점
- "동의"가 아닌 **"최고 품질의 반박"**을 검색 — 에코체임버 시대의 해독제
- 비판적 사고를 강화하는 실용 도구
- 학술, 정책, 논쟁 해결에 직결

### 기술 스택
- Elasticsearch: 시맨틱 검색, 다분야 인덱싱, 논증 구조 기반 필터링
- 데이터: arXiv 논문, 위키피디아 실험/사례, TED 토크 (오픈 데이터)
- LLM: 전제 분해, 반박 논거 구성, 조건부 분석 생성
